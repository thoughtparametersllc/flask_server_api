## Reporting a Vulnerability

We take the security of this project seriously. If you have found a vulnerability, please **do not report it via public GitHub issues**. This allows us to fix the issue before bad actors can exploit it.

### 1. Private Reporting (Recommended)
We encourage you to use GitHub's **Private Vulnerability Reporting** feature:
1.  Go to the **Security** tab of this repository.
2.  Click on **"Report a vulnerability"** to open a private advisory.
3.  This allows us to collaborate on a fix securely.

### 2. Email Reporting
If you prefer email or cannot use the private reporting tool, please send details to:
**jason.miller@thoughtparameters.com**

## What to Include
To help us triage and fix the issue quickly, please include:
* A description of the vulnerability.
* Steps to reproduce the issue (or a Proof of Concept script).
* The version of the software where you found the issue.
* Any relevant logs or configurations.

## Response Timeline
We are committed to resolving security issues promptly.
* **Acknowledgment:** We will respond to your report within **[48 hours]**.
* **Assessment:** We will confirm the issue and determine its severity.
* **Fix:** We will release a patch or mitigation as soon as possible.
